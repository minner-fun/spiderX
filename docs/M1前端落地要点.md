# M1 前端落地要点

> 把高保真定稿（`design_handoff_hifi/`）落进 `frontend/`。**唯一视觉事实来源**：`design_handoff_hifi/README.md`（精确 token）+ `frontend/src/theme/tokens.css`（已按其实现）。本文给 M1 的范围、组件清单、构建顺序、后端契约。

视觉风格：深色控制台「Mission Control」。三套字体职责分明——**Space Grotesk**(标题/大数字) · **JetBrains Mono**(指标/ID/cron/日志/域名/标签) · **Inter**(正文/按钮)。磷光青 `#4cc9e0` **只**用于交互/选中/品牌，**绝不**与健康三态色混用。

## 一、M1 范围（先做这 5 屏 + 外壳）

| 屏 | 文件 | 后端契约 |
|---|---|---|
| **登录** Login | `views/Login.vue` | M1 先做 UI + 假登录(本地 token)；真 SSO/2FA 后置 |
| **App Shell** | `layouts/AppShell.vue` | 侧栏(监控组/管理组+计数徽章) + 顶栏(项目切换/env/⌘K/巡检状态/告警铃) |

## 二、组件清单（先抽这些原子组件，全屏复用）

放 `frontend/src/components/`：
- **StatusDot** `props: state(red/yellow/green/unknown|cyan), live?` → 8px 圆点 + `box-shadow:0 0 8px <dot>`，live 加 `hfpulse`。
- **HealthBadge** `props: status` → 健康三态徽章（点+文字+暗底+暗边，色取 `--{state}-*`）。含 `unknown` ⚪。
- **ExecBadge** `props: status` → 执行态徽章（running 绿脉冲 / paused 灰 / failed 红 / disabled 弱）。
- **Panel** → 面板容器（`--bg-panel` + `--bd-panel` + `--r-panel`，含可选标题行 + 右侧 mono 副标题）。
- **KpiCard** → 顶部 2px 态色条 + label(mono) + 大数字(Space Grotesk 26–38px) + sub。
- **Pill / FilterPill** → 胶囊筛选（选中青边青底 `#0f1c20`/`#4cc9e0`，未选灰）。
- **CtaButton** `variant: accent/ghost/danger` → accent 青底辉光、ghost 描边、danger 红。
- **Sparkline / MiniBar** → ECharts 封装（深色主题，态色描边）。
- **DataTable** → 表头(mono 小字) + 行 hover `--bg-inset` + 可点击行。

图表（热力图/gantt/柱/曲线/sparkline）一律 **ECharts**，深色主题对齐 token；设计稿里的 SVG/div 只是示意。

## 三、构建顺序（建议）

1. **theme + 原子组件**：`tokens.css`(已就位) → StatusDot / HealthBadge / ExecBadge / Panel / KpiCard / CtaButton / Pill。先做一个 `views/Styleguide.vue` 把它们排出来核对像素。
2. **AppShell**：侧栏（品牌菱形 logo + ARACHNE/CRAWL CONTROL → 监控组[巡检分诊/实时·节点/告警·反爬] → 管理组[爬虫/调度/数据·规则/作者·权限] → 底部值班用户）+ 顶栏。路由高亮 = 当前路由所属导航组（detail/versions/rules/wizard 归"爬虫"组）。导航项带红/黄计数徽章。
3. **Login**：左品牌/实时 mini 控制台（网格底 `.hf-grid` + 角落辉光 + 标语"2000 个爬虫，一眼看清谁挂了、谁没数据" + 扫描线 `hfscan` + 三态计数 + 滚动日志）；右表单 480px（SSO 飞书/企业微信 + 账号 mono/密码显隐 + 保持登录 + env + "进入控制台 →" 青 CTA）。M1 假登录：填任意 → set 本地 token → 进 `/triage` 或 `/spiders`。
4. **Spiders 列表**：左筛选栏（项目/域·健康态·执行态·负责人 复选）+ 主表（名称·域·负责人·**执行态**徽章·**健康**灯·调度 cron(mono)·上次·成功率）。**关键叙事**：保留"成功率 99% 但健康 🔴"的行 + 底部黄条"跑成功率 ≠ 出数据"。行点击进详情。右上"+ 新建爬虫"青 CTA（M1 先占位，向导在 M2）。
5. **SpiderDetail**：头部卡(名称+元信息+健康徽章+执行徽章+操作[验证修复/编辑规则/运行一次]) → **分层健康信号表**(L1–L4×[层/信号/当前值/判定]，含 ⚪unknown 行 + 结论红条) → 数据量基线柱图(末柱断崖红) → 右栏(元信息[价值 P0+贡献% · 负责人 · 标签 · 线上版本→ · 数据源] / 最近运行[跑成功但新增0 陷阱] / 验证修复卡)。
6. **Versions**：左版本历史(v7 带 ●线上 绿辉光 + 作者/时间/change_msg) + 右配置 diff(mono, add绿/del红/chg黄/ctx灰) + "↶回滚"青按钮 + 顶部黄条"根除复制文件当版本"。

```

/triage                     巡检分诊（占位，M4）

/spiders/:id                爬虫详情

/spiders/:id/rules          规则编辑器（M2）

```
侧栏选中按导航组；detail/versions/rules 高亮"爬虫"组。未登录 guard → /login。

## 五、后端契约对照（M1 需要后端补的）

M0 已有：`GET /api/projects`、`GET/POST /api/projects/:pid/spiders`、`POST /api/spiders/:id/run`、WS `/ws/triage`。

M1 后端要补：
- `GET /api/spiders/:id` — 详情（含 owner、health_status、priority、contribution_pct、is_core、current_version、最近 Run 的 signals）
- `PATCH /api/spiders/:id` — enable/pause/disable、改 priority
- `GET /api/spiders/:id/versions` / `GET /api/versions/diff?from=&to=` / `POST /api/spiders/:id/rollback`
- `GET /api/spiders/:id/runs` — 最近运行（暴露"跑成功但新增0"）
- 列表接口加筛选/分页：`?status=&health=&owner=&domain=&q=&page=`
- 健康字段映射：`Spider.health_status`(structural_fail/data_dry/healthy/unknown) → 前端 HealthBadge；四层信号 `Run.signals{http_status,list_rows,field_fill_rate,dedup_new,...}`(null→⚪)。

> M1 阶段真实四层信号还没有引擎产出（M2/M3 才有），详情页/列表的健康先用占位/unknown 渲染，但**组件和契约要按真实结构写好**，M3 接上即可。

## 六、必须补的状态（设计稿未画）
- **空态**：列表无数据、详情无版本/无运行。
- **加载态**：骨架屏或低调 spinner（克制，符合控制台气质）。
- **错误/断线**：API 失败提示；WS 断线自动重连 + "重连中"标。
- **校验**：登录表单校验；查重未过禁用下一步（M2）。
- 失败/异常一律走 🔴 体系，不另造红色。
