# 交接：SpiderX（Arachne）高保真前端

## 概述
分布式爬虫管理平台的**高保真视觉设计**——深色控制台（"Mission Control"）。覆盖登录 + 12 个产品屏，是 `SpiderX设计纲要-v1.md` 的可视化落地。占位数据贯穿**招投标 + IC 元器件**双域。

## 关于这些设计文件
本包内 `*.dc.html` 是用 HTML 做的**设计参考稿**——表达最终视觉与交互意图，**不是要直接搬进生产的代码**。任务是：**用本仓已锁定的前端环境把这些设计 1:1 重建**——`Vue 3 + TS + Vite + Pinia + ECharts`，深色控制台主题（见纲要 §1）。不要照抄 HTML/内联样式，要落进 `frontend/src/theme` 的 token + 组件体系。

> 打开方式：`*.dc.html` 同目录需有 `support.js`，浏览器直接打开即可点击导航。

## 保真度：**高保真（hifi）**
最终配色、字体、间距、辉光、动效都已定稿，请**像素级还原**，只是把实现换成 Vue 组件 + ECharts 图表。下面的设计令牌是**唯一事实来源**。

## 交互 & 状态
- **路由**：主应用 12 屏单页切换；Vue 用 vue-router，侧栏高亮 = 当前路由所属导航组（detail/run/versions/wizard 归"爬虫"；rtA 归"实时·节点"）。
- **可点流转**：列表行→详情；详情→运行/版本/规则；列表"新建"→向导；实时 A↔B 顶部互切；核心站卡/队列条→详情。
- **本地交互态**：密码显示/隐藏、保持登录复选（登录）；筛选复选、tab 切换（应用）——用 Pinia 或组件局部 state。
- **实时**：纲要定义 `/ws/triage`、`/ws/runs/:id/logs`、`/ws/dashboard`。LIVE 灯脉冲、日志自动滚动、事件流增量插入。
- **动效**：状态灯 `hfpulse` 1.5s；登录扫描线 `hfscan` 5s；CTA hover 辉光增强；行 hover 提亮。时长保持 ≤ 1.5s 的交互反馈。
- **空/加载/错误态**（设计稿未画，实现需补）：列表空态、WS 断线重连提示、表单校验（查重未过禁用下一步已示意）、运行失败态用 🔴 体系。

## 与后端契约（见纲要 §4–§5）
健康三态直接映射 `Spider.health_status`(structural_fail/data_dry/healthy/unknown)；执行态 `Run.exec_status`；四层信号 `Run.signals{http_status,list_rows,field_fill_rate,dedup_new,...}`（null=unknown→⚪）；价值 `priority`(手填)+`contribution_pct`(自动)+`is_core`(派生)；🟡 闭环 `SnoozeState`；对账 `ScheduleReconcile{stuck,dlq_count}`；作者 `AuthorMetric.effective_sites`。
